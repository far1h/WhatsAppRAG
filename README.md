# WhatsApp RAG

This is a small learning project for information retrieval over exported WhatsApp chats.

The goal is simple: instead of pasting a huge chat export into ChatGPT, this app parses the chat, chunks it into conversation sessions, stores searchable embeddings, and lets me ask questions over the history locally. The current dataset is personal WhatsApp text from my ex, so the raw chat files and generated knowledge base are intentionally ignored by git.

## File Structure

- `app.py` - Gradio chat interface.
- `whatsapp_rag/whatsapp.py` - WhatsApp export parser and conversation chunking.
- `whatsapp_rag/ingest.py` - Builds summarized chunks and stores embeddings in Chroma.
- `whatsapp_rag/answer.py` - Retrieves relevant chunks and answers questions.
- `knowledge-base/chat/` - Local WhatsApp exports copied into the knowledge base.
- `_chat.txt` - Local raw WhatsApp export.
- `preprocessed_db/` - Local Chroma vector database generated by ingestion.

## Setup

Install dependencies and build the vector database with uv:

```bash
uv run python -m whatsapp_rag.ingest
```

Then run the app:

```bash
uv run app.py
```

## Environment

Create a local `.env` file with the model keys you want to use.

For Qwen through DashScope:

```bash
DASHSCOPE_API_KEY=your_key_here
CHAT_MODEL=dashscope/qwen-plus
```

Embeddings use DashScope automatically when `DASHSCOPE_API_KEY` is set and `OPENAI_API_KEY` is not set. You can also configure it explicitly:

```bash
EMBEDDING_PROVIDER=dashscope
EMBEDDING_MODEL=text-embedding-v4
```

If your DashScope account requires a region or workspace-specific compatible API base, also set `DASHSCOPE_API_BASE` or `EMBEDDING_API_BASE`.

## Privacy

Do not commit raw chat exports, generated knowledge base files, `.env`, or vector databases. They are local-only project data.
